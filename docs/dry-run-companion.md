## 2. Per-Scene 详细指挥卡

### 🎬 场景 A · Word 内生成段落（2:30）

**开口第一句**（不要冷场）：
> "我现在在 Word 里写一篇关于司美格鲁肽的综述。光标停在这里，要写第二段。"

**输入到 Copilot 框**（**逐字预先记牢**，别现场打字打错）：
```
写一段关于司美格鲁肽用于 NASH 的最新进展，引用近 2 年 PubMed 文献，使用临床综述模板。
```

**等待期话术**（覆盖 8–12 秒静默）：
> "Orchestrator 识别为写作任务 → 委派 Writer Agent → 并行调 PubMed + 加载综述模板 → GPT-4o 生成 + 强制角标引用。"

**成功判据**：
- ✅ 输出段落 3–5 段
- ✅ 出现至少 2 个角标 `[1][2]`
- ✅ 末尾有 References 列表

**如果失败 → 见 §3 故障预案 F-A1 / F-A2**

---

### 🎬 场景 B · PDF 翻译（2:30）

**开口**：
> "第二个场景：翻译。这是一份 NASH 临床试验摘要，客户要中文版，用他们的标准术语。"

**操作**：把 `abstract-semaglutide-nash.pdf` **从桌面拖到 Copilot 输入框**（不是上传按钮，拖更帅）。

**输入**：
```
把这份 PDF 翻译成中文，使用 WHODrug-CN 术语库
```

**等待期话术**（约 20 秒）：
> "Translator 同时做 3 件事：DocIntel 解析 → AI Search 召回术语 → LLM 翻译并锁定术语。这次召回了 23 个术语包括 NASH、semaglutide、HbA1c。"

**成功判据**：
- ✅ 输出包含"司美格鲁肽"，**不出现"索马鲁肽/塞马鲁肽"**
- ✅ HbA1c 保留为 HbA1c（不译为"糖化血红蛋白"）
- ✅ docx 链接可点开

**点开 docx 后的指法**：
> "标题层级保留 ✓ 段落保留 ✓ 数字单位保留 ✓ 术语一致 ✓"
> "**质量等同外包，时间 30 秒，成本几乎为零。**"

---

### 🎬 场景 C · 合规审查（2:00）

**开口**：
> "最后一个场景，最关键的：合规审查。市场部同事写的初稿。"

**操作**：Ctrl+V（你已经预置好剪贴板）+ 输入：
```
审查这段药品介绍是否合规
```

**等待期话术**（约 15 秒）：
> "Compliance 并行做两件事：规则审查（5 条 FDA/NMPA 警语，每条 GPT-4o 判定）+ 事实审查（抽数字声明，自动从 PubMed 拉证据对照）。"

**逐条指点的台本**（**节奏控制在 60 秒内**）：

1. 🔴 Critical："对阿尔茨海默病有效，建议医生处方" → "off-label 推广，**这条业务最在意**"
2. 🔴 High："最佳选择"、"根治" → "绝对化用语"
3. 🔴 High："安全性良好，几乎无副作用" → "未公平披露安全信息"
4. 🟡 High：**"风险降低 50%"** → "**事实错误，AI 自动查了 PubMed 上 SUSTAIN-6 原文，实际 HR 0.74 = 26%**" ← **本场 demo 王炸**，停一秒
5. 🟡 Medium："大幅优于所有 SGLT-2 抑制剂" → "未基于头对头试验"

**点 Accept 演示**（仅点第 1 条，别全点，时间不够）：
> "建议替换文本已进剪贴板。这是合规专家半天的活儿。**15 秒完成。**"
