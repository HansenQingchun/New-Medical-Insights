### 🎬 [00:30–03:00] 场景 A：Word 内生成段落（核心高光时刻）

**[操作]** 打开预先准备好的 Word 文档（标题：《司美格鲁肽研究综述（草稿）》），文档已写了背景段，光标停在第二段空白处。

**[操作]** 在 Word 右侧打开 **Microsoft 365 Copilot** 侧边栏 → 切到 **NMI** 智能体。

> "我现在在 Word 里写一篇关于司美格鲁肽的综述。第一段已经写好背景，第二段我想要最新的临床证据，但是要从 PubMed 查文献，还要正确引用 —— 这就是 2 小时的工作。"
> "看看 NMI 怎么做。"

**[操作]** 在 Copilot 输入框输入：
```
写一段关于司美格鲁肽用于 NASH 的最新进展，引用近 2 年 PubMed 文献，使用临床综述模板。
```

**[等待 ~10 秒，流式输出]**

> "NMI 的 **Orchestrator** 识别到这是一个写作任务，**委派给 Writer Agent**。"
> "Writer Agent 同时做两件事 —— 调 PubMed 检索文献，加载我们预置的'临床综述模板'。"
> "然后用 GPT-4o 生成内容，**每个临床数据点强制加角标引用**。"

**[结果出现：4 段式段落 + 角标 + References 列表]**

**[操作]** 鼠标悬停在 `[2]` 角标上 → 显示文献摘要悬浮卡片。

> "每个引用可以悬浮预览，点击跳转到 PubMed。这不是在 ChatGPT 里 copy-paste —— 这是在我的 Word 文档里，引用真实文献。"

**[操作]** 点击 **"📥 插入到 Word"** 按钮 → 段落落入光标位置。

**[操作]** 鼠标点击文档中段落的角标 `[1]` → 自动跳到文末 References。

> "段落和参考文献已经进了文档，引用编号已经对齐到我前面段落的列表里。"
> "这一段，**原本 2 小时，现在 15 秒**。"

---

### 🎬 [03:00–05:30] 场景 B：PDF 翻译 + 术语锁定

**[操作]** 切到浏览器，打开 Copilot Studio Web 测试通道（或继续用 Word Copilot 也可以）。

> "第二个场景：翻译。这是一份 NASH 临床试验的英文摘要 PDF（场景 B 用的 abstract-semaglutide-nash.pdf）。客户要中文版，要用他们的标准医学术语库 —— '司美格鲁肽'必须是这个译法，不能是'索马鲁肽'或'塞马鲁肽'。"

**[操作]** 拖拽 PDF 到 Copilot 聊天框，输入：
```
把这份 PDF 翻译成中文，使用 WHODrug-CN 术语库
```

**[等待，过程中讲解]**

> "Translator Agent 在做几件事："
> "1. **Document Intelligence** 解析 PDF 的段落和结构；"
> "2. 对每段在 **Azure AI Search** 的术语库里召回命中术语 —— 这次命中了 23 个，包括 NASH、semaglutide、HbA1c 等；"
> "3. 然后调 LLM 翻译时**把这些术语锁定**给模型，禁止变体。"

**[结果出现：翻译预览 + 下载链接]**

**[操作]** 点击下载链接 → 打开 docx 文件。

> "看，标题层级、段落、数字、单位都保留了。司美格鲁肽 4 处，全部一致；HbA1c 没翻成'糖化血红蛋白'是因为术语库里指定保留缩写。"
> "**质量等同外包，时间 30 秒，成本几乎为零。**"

---

### 🎬 [05:30–07:30] 场景 C：合规审查

**[操作]** 回到 Copilot 聊天框（或 Teams 也可以，演示三端体验）。

> "最后一个场景，可能是最关键的：合规审查。"
> "我手上有一段药品介绍材料，是市场部同事写的初稿。我想知道 —— 是不是符合 FDA Fair Balance？有没有事实错误？"

**[操作]** 粘贴 `compliance-test-paragraph.md` 中的中文段落，输入：
```
审查这段药品介绍是否合规
```

**[等待 ~15 秒，流式输出]**

> "Compliance Agent **并行做两件事**："
> "1. **规则审查** —— 加载我们预置的 5 条 FDA/NMPA 警语规则，每条规则用 GPT-4o 判定是否违反；"
> "2. **事实审查** —— 抽取段落中所有数字声明，自动从 PubMed 拉证据对照。"

**[结果出现：高亮列表 + 6 条 findings + Accept/Ignore 按钮]**

**[操作]** 逐条快速指：
1. 🔴 **Critical**：'对阿尔茨海默病有效，建议医生处方' → off-label 推广（**这条最关键，业务最在意**）
2. 🔴 **High**：'最佳选择'、'根治' → 绝对化用语
3. 🔴 **High**：'安全性良好，几乎无副作用' → 未公平披露安全信息
4. 🟡 **High**：'风险降低 50%' → **事实错误**，SUSTAIN-6 实际是 26%
5. 🟡 **Medium**：'大幅优于所有 SGLT-2 抑制剂' → 未基于头对头试验

> "每条都标了**严重度**和**置信度**，告诉了我违反哪条规则、为什么违规、建议怎么改。"
> "尤其是第 4 条 —— **事实错误，AI 自动查了 PubMed 上的 SUSTAIN-6 原文核对**，告诉我实际是 HR 0.74，相对风险下降 26%，不是 50%。"

**[操作]** 点击第 1 条的 "ACCEPT" 按钮 → 建议替换文本到剪贴板。

> "这是合规专家半天的活儿。现在 15 秒。"

## 🎯 常见 Q&A（预演练）

| 问题 | 回答要点 |
|---|---|
| 数据安全？是否会用我们的数据训练模型？ | 不会。Azure OpenAI 与 AI Foundry 的企业部署**默认不用客户数据训练**。所有数据驻留在客户租户的 Azure 区域内。 |
| 与 ChatGPT Plus / 直接调 OpenAI 有什么区别？ | 一个 AI 助手，三端原生入口；企业级身份、审计、术语库、规则库都是企业可控的；**Microsoft 兜底 SLA 与合规承诺**。 |
| 能不能换成 GPT-5 或 Claude？ | 可以。Azure AI Foundry 支持 GPT-4o、GPT-5、Claude on Foundry、Mistral 等多模型，可按场景路由。 |
| 多大规模？支持多少用户？ | Copilot Studio 按消息计费，Azure AI Foundry 按 token 计费，弹性扩展。Demo 这套架构可直接支持 1000+ 并发用户。 |
| 实施周期？ | Demo 已实现是 6 周；生产化首期 3 个月（含数据准备、规则定制、用户培训）。 |
| 价格区间？ | 取决于用户数与调用量，建议会后做 ROI 测算。整体 TCO 通常比自建 + 外包翻译低 40% 以上。 |
| 我们的术语库已经有 5 万条，能直接导入吗？ | 可以。AI Search 单索引支持千万级条目，Demo 已演示 50 条，生产环境 5 万条秒级检索。 |
| 中文医学术语标准化怎么处理？ | 支持 WHODrug、MedDRA、ICD-10/11，可与客户已有标准对接。 |
